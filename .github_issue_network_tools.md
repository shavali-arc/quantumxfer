## üéØ Overview

Add Network Diagnostic Tools to QuantumXfer for DNS lookup, port scanning, PING, traceroute, and other network utilities directly from terminal sessions.

## üí° Problem Statement

DevOps engineers troubleshooting network issues need to:
- Check DNS resolution
- Scan open ports
- Test connectivity
- Trace routes to hosts
- Check domain info

**Challenge**: Must use separate tools or open new terminal windows (nslookup, nmap, ping, traceroute, whois)

## ‚ú® Solution

Network tools integrated in terminal:
- DNS lookup & verification
- Port scanning
- Ping with statistics
- Traceroute
- WHOIS lookup
- IP information

## üìã Requirements

### Core Functionality
- [ ] DNS lookup (A, AAAA, MX, TXT records)
- [ ] Reverse DNS
- [ ] Port scanning (check open ports)
- [ ] PING with statistics
- [ ] TRACEROUTE to destination
- [ ] IP address information
- [ ] Domain WHOIS lookup
- [ ] Network interface info

### Terminal Commands
```bash
# DNS lookup
NSLOOKUP example.com
NSLOOKUP -type=MX example.com
NSLOOKUP -type=TXT example.com

# Reverse DNS
NSLOOKUP -reverse 8.8.8.8

# Port scan
PORTSCAN host.internal --ports=22,80,443,3306,5432
PORTSCAN -p 1-1000 host.internal

# Ping
PING host.internal --count=5

# Traceroute
TRACEROUTE example.com

# IP information
IPINFO 192.168.1.1

# Domain WHOIS
WHOIS example.com

# Network interfaces
IFCONFIG

# Route info
ROUTE example.com
```

### UI/UX Requirements
- [ ] Clean result formatting
- [ ] Color-coded status (open/closed ports)
- [ ] Response time display
- [ ] Record type highlighting
- [ ] Error messages clear
- [ ] Success/failure indicators
